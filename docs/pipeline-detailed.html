0

Directory Discovery

Find funder URLs from grant aggregator websites and the charities database. Runs in parallel with Step 1 — results not needed until Step 3.

Tavily 0a — Directory Page Extraction

The NZ market has 12 hardcoded grant directory URLs (e.g. communitymatters.govt.nz, philanthropy.org.nz, generosity.org.nz, sportnz.org.nz). Each page is extracted via Tavily and scanned for outbound links — every external URL is collected as a potential funder.

This is a brute-force link harvesting step. Many of the collected URLs won't be grant pages (they might be sponsor logos, partner links, etc.), but that's fine — the isGrantPage() filter in Step 3 will remove irrelevant ones before extraction.

Example

Extracting https://www.philanthropy.org.nz/our-members (a real directory in our config) returns page content containing links to individual funders like:
https://www.foundationnorth.org.nz, https://www.trustwaikato.co.nz, https://ratafoundation.org.nz
Similarly, extracting https://tindall.org.nz/other-funding-resources/ yields links to funders Tindall recommends.
These become discovered funder URLs.

Serper 0b — Exhaustive Directory Site Search

For each of the 12 directory domains, runs multiple site:domain Serper queries to mine grant listings deeper than the homepage:

  • Base queries: site:communitymatters.govt.nz grants apply, site:communitymatters.govt.nz grants 2026, etc. (6 per directory)
  • Sector queries: site:sportnz.org.nz Children & Young People grants (2 per sector)
  • Region queries: site:philanthropy.org.nz Auckland grants (2 per region)
  • Sector × region: site:generosity.org.nz Auckland Community Development grants (up to 6 sectors × 4 regions)

Queries are deduplicated before execution. Typically produces 30-60 unique queries per directory.

DB Charities Register Lookup

SQLite full-text search on a local copy of the NZ Charities Register. Matches charities by sector + region + funding purpose keywords. Returns two categories:

  • Enriched funders (have grantSummary field) — these skip Tavily entirely and are injected as synthetic pages at Step 3d. Zero extraction cost.
  • Unenriched funders (URL only) — their grant URL is added to the extraction list for Tavily processing.

DB URLs are capped: only fills remaining capacity after directory-discovered URLs, up to a total of 200 URLs.

GPT-4o-mini Regional Query Generation

A single GPT-4o-mini call generates 20-30 search queries designed to find small and obscure local funders that wouldn't appear in a national enumeration. The prompt asks for queries covering:

  1. Community and regional trusts
  2. District and city councils
  3. Small/local charitable trusts
  4. Private and family foundations
  5. Corporate and law firm trusts
  6. Sector-specific local grants
  7. Sub-regional areas (towns, districts within each region)
Example Regional Queries (for Auckland + Waikato)

"Auckland community trust grants apply 2026"
"Hamilton city council community fund apply"
"Waikato family foundation grants nonprofits"
"North Shore charitable trust community grants"
"Matamata Piako district council grants"

Cost for this step

Tavily: 12 directory pages × $0.008 = $0.10 • Serper: ~200-400 queries × $0.001 = $0.20-0.40 • GPT-4o-mini: 1 call = <$0.01

1

Funder Enumeration

Ask GPT-4o to recall every grant-giving organisation it knows, organised by category. This is a knowledge recall task — no web searching.

Results from all 7 category calls are merged. Funders are deduplicated by lowercased name (first-seen wins). Typically produces ~170 unique funders.

REGION FILTER Enumerated funders filtered to user's selected regions + national/nationwide/unspecified. For an Auckland+Waikato search, this removes councils from Southland, trusts from Canterbury, etc. Typically cuts ~50% of funders. Cost for this step

GPT-4o: 7 calls (4 category + 1 programme + 1 org extract + 1 Tavily) = ~$0.80-1.50 depending on token usage

2

Multi-Source Web Search

Search the web for grant application pages using 7 different strategies simultaneously. Every URL found is tagged with its origin for diagnostics.

All search tasks run concurrently (up to 20 parallel Serper connections). Results are pooled into a single rawSearchHits[] array. Each Serper query returns ~10 results.

Serper Enum Searches origin: 'enum'

One Serper query per enumerated funder/programme. Uses the searchQuery generated by GPT in Step 1.

This is typically the highest-yield source — produces the most kept grants per URL. The query is highly targeted: it contains the exact funder name + "grants apply" + the current year.

Example Queries (using real curated funders from nz.ts)

"Pub Charity grants apply New Zealand 2026" → finds pubcharitylimited.org.nz/grants (curated URL in config)
"Foundation North funding apply 2026" → finds foundationnorth.org.nz/funding (curated, regions: Auckland, Northland)
"Auckland Council community grants apply 2026" → finds aucklandcouncil.govt.nz/grants/.../default.aspx (curated, regions: Auckland)
"NZCT grants apply New Zealand 2026" → finds nzct.org.nz/grants/ (curated, national)

Serper Seed Queries origin: 'seed'

Static, hardcoded queries from the market config's seedQueryTemplates (8 templates in NZ config). These are broad, market-wide searches designed to catch funders that GPT didn't enumerate. Templates use placeholders: {country}, {purpose}, {year}, {currency}.

Actual NZ Seed Query Templates (from nz.ts)

"New Zealand grants apply "$10,000 for an IT systems upgrade" non-profit 2026"
"New Zealand community charitable trust grants 2026 apply"
"New Zealand local council community grants apply 2026"
"site:communitymatters.govt.nz lottery grants apply"
"New Zealand corporate foundation community grants apply 2026"
"New Zealand government grants not-for-profit 2026"
"NZ gaming trust grants apply 2026"
"pokie trust grants New Zealand community apply 2026"

Serper Regional Searches origin: 'regional'

The 20-30 GPT-generated queries from Step 0 are executed here. These specifically target small/local funders in the user's operating regions — community trusts, district councils, family foundations.

Serper Directory Deep-Dive origin: 'directory-deepdive'

For each of the 12 grant directories, runs a site:domain search scoped to the user's funding purpose. This finds sector-specific pages within directories that the Step 0 broad crawl may have missed.

Example

"site:communitymatters.govt.nz IT systems upgrade community"
"site:perpetualguardian.co.nz IT systems upgrade community"

GPT-4o-mini Serper Purpose Seeds origin: 'purpose'

GPT-4o-mini generates 10 highly-targeted queries based on the org's specific funding purpose and website content. These find niche programmes the general enumeration would miss.

The prompt instructs GPT to vary by approach: sector, activity type, population served, funding mechanism.

Example Purpose Seed Queries (for IT upgrade)

"New Zealand technology upgrade grant community organisations 2026"
"digital capability fund nonprofits NZ apply"
"IT infrastructure grants charitable organisations New Zealand"

GPT-4o-mini Serper Associative Gap-Fill origin: 'associative'

After enumeration, GPT analyses the funder list and identifies gaps — funders that should exist but weren't listed. It looks for:

  • Regional peers: If a community trust in one region is listed but the equivalent in the org's region isn't
  • Peers of previous funders: Similar funders to ones the org has been funded by before
  • Sector-specific trusts: Dedicated health/education/environment trusts missing from the list
  • Small/local trusts: Family foundations, community endowments too niche for enumeration
  • Recently rebranded organisations

Generates 10-15 targeted queries. From the diagnostics, this source produces the highest average scores (7.7 in our test) despite low volume — it finds high-quality matches the other steps miss.

Serper Broad Category Searches origin: 'broad'

Automated query generation that combines grant terminology × sectors × regions. No GPT involved — purely algorithmic. Covers:

Uses 7 hardcoded grant terminology variations from generateBroadCategorySearches():

  • "grants apply", "funding opportunities", "community grants", "charitable trust grants", "grant applications open", "funding available nonprofits", "community fund apply"

These are combined with sectors, regions, and purpose keywords:

  • Sector × region: "Auckland New Zealand Children & Young People grants apply", "Community Development funding Waikato New Zealand nonprofits"
  • Region-wide: "Auckland New Zealand grants apply", "Waikato New Zealand community grants"
  • Purpose keywords: individual words >4 chars from the funding purpose: "systems grants New Zealand apply", "upgrade funding New Zealand nonprofits"
  • Grant types: 5 fixed types: "New Zealand project grants nonprofits apply", "New Zealand operational funding nonprofits apply", "New Zealand capital grants nonprofits apply", "New Zealand programme funding nonprofits apply", "New Zealand capacity building grants nonprofits apply"

Typically generates 60-100 unique queries.

Cost for Step 2

Serper: ~200-400 queries × $0.001 = $0.20-0.40 • GPT-4o-mini: 3 calls (purpose + regional + associative) = <$0.02

2b

Gap-Fill Search

Second-pass analysis — identify which funder categories are still underrepresented in the search results and run targeted queries to fill the gaps.

GPT-4o-mini Serper Gap Analysis origin: 'gap-fill'

GPT-4o-mini receives a list of all domains found so far and identifies which funder categories are underrepresented. It then generates targeted queries to fill those gaps.

For example, if no URLs from gaming trusts have been found, it might generate "NZ gaming trust grants community apply 2026".

2c

Per-Domain Site Crawl

Many funder searches return homepages. Crawl within each discovered domain to find grant/apply pages buried deeper. Currently DISABLED via toggle.

Serper Domain Crawl origin: 'site-crawl'

For each unique domain found in rawSearchHits (excluding known non-grant domains), runs 4 Serper queries:

  • site:foundationnorth.org.nz grants apply
  • site:foundationnorth.org.nz funding apply
  • site:foundationnorth.org.nz community grants
  • site:foundationnorth.org.nz grant application

These are the 4 fixed query patterns from crawlFunderSites() in route.ts (line ~911). Every domain gets the same 4 queries.

This is the most expensive search step by volume. Before optimisation, it ran ~2,600 Serper queries and produced ~5,900 raw URLs but only ~18 kept grants (10% of total). Now controlled by two filters:

  1. Skip already-capped domains: Domains with 3+ URLs already found are skipped — any new URLs would be discarded by the per-domain cap anyway.
  2. Cap at 200 domains: Only crawls the first 200 qualifying domains (ordered by search relevance).
Toggle

Site crawl is controlled by ENABLE_SITE_CRAWL at the top of route.ts. Currently set to false. When disabled, logs "Step 2c: Site crawl DISABLED" and skips this entire step.

▼

Deduplication, Merge & Filtering

Reduce thousands of raw URLs down to a manageable extraction list. Four filtering stages, each with a specific purpose.

Stage 1: URL Deduplication

deduplicateByUrl(rawSearchHits) — normalises each URL (strips www., lowercases, removes trailing slashes) and keeps only the first occurrence. The origin tag of the first-seen hit is preserved for diagnostics.

Normalisation Example (using normaliseUrl() from route.ts line ~423)

Strips www., lowercases, removes trailing slashes:
https://www.FoundationNorth.org.nz/Funding/ → foundationnorth.org.nz/funding
https://foundationnorth.org.nz/funding → same key → deduplicated

Curated URL https://www.foundationnorth.org.nz/funding (from nz.ts line 148) and a Serper hit for https://www.foundationnorth.org.nz/funding/ normalise to the same key. The curated version wins because curated URLs are first in the merge order.

Stage 2: Three-Source Merge

Three URL sources are merged into a single list, in this priority order:

  1. Curated URLs (~200 entries from nz.ts curatedFunderUrls) — hand-verified funder pages with optional regions tags, filtered to user's selected regions. Examples: nzct.org.nz/grants/ (national), foundationnorth.org.nz/funding (regions: Auckland, Northland), trustwaikato.co.nz/funding (regions: Waikato). National funders (no regions tag) always included.
  2. Discovered URLs (Step 0) — funder URLs from directory pages + DB unenriched funders.
  3. Search hits (Step 2) — URLs from all 9 search strategies.

Priority matters because deduplication keeps the first occurrence. If a URL appears in both curated and search hits, it gets origin 'curated'.

Stage 3: Grant Page Filter

isGrantPage(url) removes URLs that are categorically never grant application pages. The filter combines NON_GRANT_PATH_PATTERNS (~30 regex patterns, route.ts line ~262) with the market's excludedDomains list:

  • News/media: /press-release/, /media-release/, /blog/
  • Auth: /login, /signin, /register
  • Legal: /privacy-policy, /terms-of-use
  • Jobs: /careers/, /vacancies/, /jobs/
  • Excluded domains (NZ config): nzherald.co.nz, stuff.co.nz, rnz.co.nz, scoop.co.nz, wikipedia.org, linkedin.com, facebook.com, etc.

Curated URLs bypass this filter entirely — they're hand-verified as grant pages.

Stage 4: Per-Domain Cap

MAX_PER_DOMAIN = 3 — maximum 3 URLs per domain are sent to extraction. This prevents one funder's site from consuming a disproportionate share of the Tavily budget.

URLs are processed in merge order (curated first), so if a domain has 2 curated URLs and 5 search hits, only the 2 curated + 1 search hit are kept.

Typical Funnel (Auckland + Waikato search)

~3,500 raw search hits → ~1,100 after dedup → ~900 after grant filter → ~800 after per-domain cap

3

Content Extraction

Extract full page content from every URL that survived filtering. Four extraction layers with progressive fallbacks. This is the most expensive step.

Tavily 3a — Primary Extraction

All URLs in allUrlsToExtract are sent to Tavily in batches of 20, with 8 concurrent batch connections. Tavily renders the page (including JavaScript), extracts the raw text content, and returns up to 8,000 characters per page.

Typical success rate: ~70-80%. Failures happen due to bot blocking, JavaScript-heavy SPAs, server timeouts, or CAPTCHA walls.

Serper 3b — Search Fallback

For URLs that Tavily couldn't extract, groups them by domain and runs site:domain grants apply funding via Serper. If the search returns results with content longer than 100 characters, those snippets become the page content.

This catches sites that block Tavily's scraper but are indexed by Google.

Free 3c — Snippet Promotion

For URLs that both Tavily and the search fallback failed on, promotes the original Serper search snippet from Step 2 (if it's longer than 200 characters). This is the last resort before giving up on a URL.

Snippets are short (~200-500 chars) but often contain enough information for GPT to identify the grant programme name, funder, and basic eligibility criteria.

DB 3d — DB Enrichment Injection

Enriched Charities Register funders (those with a grantSummary field from previous research) are injected as synthetic pages. Their content is constructed from the database fields: name + grant summary + registered purpose.

Zero Tavily cost. This is why enriching the charities database pays for itself — each enriched entry is a "free" page that doesn't need Tavily extraction.

Cost for Step 3

Tavily: ~800 URLs × $0.008 = ~$6.40 • This is typically ~80-91% of total search cost. The region filter in Step 1 and site-crawl toggle are specifically designed to reduce this number.

4

Grant Extraction

GPT reads every extracted page and identifies individual grant programmes with structured metadata. Many pages contain multiple grants.

PRE-FILTER isGrantPage(url) applied again to extracted pages. This second pass catches pages whose content reveals they're not grant-related (the first pass in the merge stage only checks URL patterns).

GPT-4o-mini Page Batch Processing

Filtered pages are batched in groups of PAGE_BATCH_SIZE = 4 and sent to GPT-4o-mini. Each batch is a single API call that receives the full content of 4 pages and must extract every grant programme it finds.

15 batches run concurrently. For 800 pages, this means ~200 batches = ~200 GPT-4o-mini calls.

The extraction prompt is detailed and contains several important rules:

  • Error pages (404, access denied) → return [] immediately
  • News articles about grants → only extract if a direct link to the actual grant page is found in the content
  • "Record of grants" (past recipient lists) → extract only the parent programme, not individual entries
  • Extract ALL grants regardless of apparent relevance — scoring will assess fit
  • Skip only: commercial-only grants, grants restricted to a different country
  • URL field: use the most specific URL available (look for hyperlinks in the page content pointing to the actual application page)
  • Include a 1,500-char verbatim excerpt of the most relevant content
Example Input (one page in a batch — using real curated URL pubcharitylimited.org.nz/grants from nz.ts line 131)

Grant types available: - Community Grants: up to $50,000 for community projects - Sports Grants: up to $100,000 for sports facilities and equipment - Health & Wellbeing: up to $30,000 for health-related community initiatives ...

Grant Provenance Tracking

During batch processing, each extracted grant is assigned the origin of its source page. The matching priority:

  1. Exact URL match: grant URL matches a URL in the origin map
  2. Domain match: grant URL's domain matches a source page in the same batch
  3. Batch fallback: most common non-unknown origin among the batch's source pages

This ensures near-zero "unknown" origin grants in the diagnostics table.

Grant Deduplication

deduplicateGrants() removes duplicate grants by normalised funder||name key. The same grant may appear on multiple pages (e.g. a directory listing AND the funder's own site). First-seen wins.

Cost for Step 4

GPT-4o-mini: ~200 batch calls × ~2K input tokens each = ~$0.06-0.12. Very cheap relative to Step 3.

5

Scoring & Ranking

GPT-4o reads every discovered grant with full org context and scores it on three dimensions. This is the quality gate.

GPT-4o Batch Scoring

Grants batched in groups of SCORE_BATCH = 25. Each batch sent to GPT-4o with:

  • Full org context (website, type, regions, sectors, purpose, amount, previous funders)
  • Org website content (first 1,500 chars)
  • Current date
  • Each grant's metadata + up to 2,000 chars of page content

15 concurrent scoring calls. The first batch also generates orgSummary.

Scoring Prompt — Pre-Scoring Checks

Before scoring, GPT applies three sequential checks:

1. URL Quality Check: If the content reads like a news article, press release, or past recipients list (rather than an actual grant page), zero out all scores. This catches grants that slipped past the extraction prompt's rules.

2. Geographic Eligibility Check: If the grant explicitly restricts applicants to a different country (e.g. "US-based organisations only"), zero out scores. International/global programmes are eligible.

3. Regional Relevance: National funders always score normally. Regional funders that don't serve the org's regions get a 3-4 point attainability reduction.

Scoring Dimensions

Dimension Weight Scale What it measures
Alignment 50% 0-3 poor | 4-6 partial | 7-8 good | 9-10 exact How well the grant's purpose AND form of support match the org's mission AND specific funding request. Includes the form-of-support check (cash vs in-kind).
Attainability 30% 1-2 very competitive | 5-6 moderate | 9-10 strong match Likelihood the org wins given competition level and eligibility fit. Regional/less-competitive funders score higher.
Ease 20% 1-2 multi-stage | 5-6 full proposal | 9-10 simple form How easy it is to apply. Simple online forms score highest; multi-stage processes with site visits score lowest.

Score Formula

overall = (alignment × 0.5) + (attainability × 0.3) + (ease × 0.2) Example: alignment=8, attainability=7, ease=6 overall = (8 × 0.5) + (7 × 0.3) + (6 × 0.2) = 4.0 + 2.1 + 1.2 = 7.3

Form-of-Support Check

Added to prevent false positives like "Digital Wings" (donated IT equipment) matching an org seeking $10,000 cash for an IT upgrade. The scorer explicitly checks:

  • Cash grants — direct monetary support
  • In-kind donations — donated goods, equipment, materials
  • Services — training, mentoring, capacity building
  • Fee waivers — reduced-cost access to products/services

If the org seeks cash but the grant provides in-kind support, alignment is capped at 3-4 regardless of topic overlap.

Deadline Rule

GPT only sets a deadline field if the page content contains a specific future date explicitly stated as a closing date. Past dates are ignored. No date stated = rolling/open cycle. This prevents false "closed" classifications.

Cost for Step 5

GPT-4o: ~5-8 batch calls × ~6K input tokens each = ~$0.30-0.80. Second most expensive step after Tavily extraction.

✓

Output & Diagnostics

Final results are returned to the frontend. Console logs provide full cost and provenance diagnostics.

API Response

The search API returns a JSON object containing:

  • grants[] — scored GrantOpportunity objects, sorted by overall score
  • orgSummary — GPT's 2-3 sentence description of the organisation
  • searchedAt — ISO timestamp
  • market — "nz" (used for currency formatting)
  • inputs — original form inputs (saved for re-running searches)
  • diagnostics — per-step provenance breakdown (see below)
